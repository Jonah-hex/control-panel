# مسح تفاصيل إنشاء مبنى سكني — اقتراح أقسام مكملة

## الوضع الحالي (الكاردات الموجودة)

| الكارد | المحتوى الرئيسي |
|--------|-----------------|
| **معلومات أساسية** | الاسم، المالك، رقم القطعة، رخصة البناء، رقم الصك، الحي، المساحة، العنوان |
| **معلومات المبنى** | الأدوار، عدد الوحدات، حالة البناء، وحدات/دور، مواقف، غرف سائقين، نوع الشارع، الاتجاه، مصاعد، سنة البناء، الوصف |
| **المرافق والتأمين** | التأمين (نعم/لا)، رقم البوليصة، رفع البوليصة، عداد الكهرباء الرئيسي، عداد المياه |
| **بيانات الحارس** | الاسم، الجوال، رقم الغرفة، صورة الهوية، الوردية، الراتب |
| **الموقع والصور** | رابط خرائط، معرض الصور |
| **اتحاد الملاك** | المدير، رقم التسجيل، الحساب، الآيبان، التواصل، وحدات مسجلة، بداية/نهاية، الرسوم، ماء/كهرباء مشمولة |
| **المكتب الهندسي** | روابط: الصكوك ومحاضر الفرز، الخرائط ومستندات المبنى |
| **عدادات الكهرباء** | جدول وحدات + رقم عداد لكل وحدة |
| **غرف السائق** | جدول وحدات + رقم غرفة سائق لكل وحدة |

---

## ما يغطيه النظام اليوم

- **هوية العقار:** اسم، عنوان، قطعة، رخصة، صك، حي، مساحة.
- **هيكل المبنى:** أدوار، وحدات، مصاعد، مواقف، غرف سائقين، اتجاه، وصف.
- **التشغيل اليومي:** حارس، مرافق، تأمين، مياه وكهرباء (أساسية).
- **الإدارة القانونية/الإدارية:** اتحاد ملاك، صكوك، محاضر فرز، مستندات هندسية.
- **ربط الوحدات:** عدادات كهرباء وغرف سائق على مستوى الوحدة.

## اقتراح أقسام جديدة لإكمال هوية التفاصيل

### أولوية عالية (تكمل صورة العمارة السكنية فعلياً)

1. **كارد «المصاعد والصيانة»**
   - **الهدف:** توثيق المصاعد وربطها بالصيانة.
   - **الحقول المقترحة:** عدد المصاعد (موجود جزئياً في معلومات المبنى)، نوع المصاعد (ركاب/بضائع)، شركة الصيانة، رقم عقد الصيانة، تاريخ آخر صيانة، رقم طوارئ المصاعد.
   - **الفائدة:** يكمّل «معلومات المبنى» و«المرافق» ويربط لاحقاً بطلبات الصيانة إن وُجدت.

2. **كارد «الرخص والتصاريح» (أو توسيع الأساسيات)**
   - **الهدف:** مكان واحد لكل ما له صلاحية زمنية.
   - **الحقول المقترحة:** رخصة البناء (موجودة في الأساسيات)، رخصة تشغيل/استخدام، تاريخ انتهاء رخصة التشغيل، تصريح الدفاع المدني (إن يُستخدم)، ملاحظات.
   - **الفائدة:** عدم تشتت الرخص بين الأساسيات والمرافق؛ جاهزية لتذكيرات انتهاء الصلاحية لاحقاً.

3. **كارد «مواقف السيارات» (تفصيلي)**
   - **الهدف:** توثيق أكثر من مجرد «عدد المواقف».
   - **الحقول المقترحة:** عدد المواقف (موجود)، مواقف سطحية/تحت أرضية، هل مواقف مرقمة/مربوطة بوحدات (نعم/لا)، ملاحظات (مثلاً: زيارة، حجز، إلخ).
   - **الفائدة:** أوضح لهوية العمارة السكنية ويمكن لاحقاً ربط وحدات بمواقف إن رغبت.

### أولوية متوسطة (تحسين الاحترافية والاستخدام)

4. **توسيع كارد «المرافق والتأمين»**
   - **إضافة:** شركة الكهرباء، شركة المياه (إن وُجدت)، رقم الاشتراك أو العقد إن أمكن.
   - **الفائدة:** اكتمال صورة المرافق والاشتراكات دون كاردات كثيرة.

5. **كارد «جهات الاتصال الطارئة»**
   - **الهدف:** أرقام واحدة للطوارئ على مستوى العمارة.
   - **الحقول المقترحة:** دفاع مدني، شرطة، شركة كهرباء طوارئ، شركة مياه، صيانة مصاعد، إدارة اتحاد الملاك (إن مختلفة عن بيانات اتحاد الملاك).
   - **الفائدة:** مكمّل لبيانات الحارس واتحاد الملاك؛ مفيد للإدارة والحراس.

### أولوية لاحقة (بعد استقرار الأساس)

6. **كارد «المستأجرون / السكن الحالي»**
   - يخص التشغيل أكثر من «إنشاء المبنى»؛ يمكن تأجيله حتى تكتمل وحدات وحجوزات ومبيعات.

7. **ربط مع طلبات الصيانة والمصروفات**
   - لا يُعد كارد «تفاصيل» جديداً، بل توسيع استخدام البيانات (مصاعد، مرافق، اتحاد ملاك) في شاشات الصيانة والمصروفات لاحقاً.

---

## ملخص التوصية

- **إضافة كاردات جديدة مقترحة في المرحلة الحالية:**
  - **المصاعد والصيانة** (أولوية عالية).
  - **الرخص والتصاريح** (أولوية عالية، أو دمجها في «معلومات أساسية» مع قسم فرعي واضح).
  - **مواقف السيارات (تفصيلي)** (أولوية عالية إن أردت تمييز العمارة بمواقف مرقمة/مربوطة).
- **توسيع دون كارد جديد:**
  - في **المرافق والتأمين:** إضافة حقول شركة الكهرباء/المياه ورقم الاشتراك إن أمكن.
- **كارد «جهات الاتصال الطارئة»** اختياري لكنه يعزز هوية النظام كـ «نظام عمارة سكنية» جاهز للاستخدام اليومي.

بهذا يصبح إنشاء مبنى سكني مغطى من (الهوية، الهيكل، المرافق، الحارس، الموقع، اتحاد الملاك، الهندسي، العدادات، غرف السائق، المصاعد/الصيانة، الرخص، المواقف التفصيلية، واختيارياً الطوارئ) دون تكرار كبير، مع إمكان توسيع لاحق لربط الرخص بالتنبيهات والصيانة بالمصروفات.

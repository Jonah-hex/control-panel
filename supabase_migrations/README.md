# هجرات Supabase

## بيئة جديدة
**لأي Supabase جديد:** نفّذ **`run_all_current_schema.sql`** مرة واحدة في SQL Editor (لصق كامل ثم Run).  
هذا الملف يجمع كل تغييرات هذا المجلد بدون تكرار؛ باقي الملفات تبقى أرشيفاً ومرجعاً.

## قبل التشغيل
يجب أن تكون الجداول `building_investors` و`unit_investments` و`dashboard_appointments` موجودة (مثلاً من `investors_schema.sql` و`create_dashboard_tasks_and_appointments.sql` في جذر المشروع).
